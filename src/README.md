# 程式碼目錄

`src/` 用來放正式、可重複執行的程式碼。探索中的程式可以先放在 `notebooks/`，確認需要重複執行後再整理到這裡。

## 建議結構

目前尚未確定論文題目與程式語言，因此先保留技術中立的結構建議：

```text
src/
├── README.md
├── data/          # 資料讀取、清理、轉換
├── models/        # 模型、演算法或核心方法
├── evaluation/    # 評估指標、比較方法、統計檢定
├── visualization/ # 圖表產生
└── utils/         # 共用工具
```

等研究方向確定後，再依實際技術建立需要的子資料夾。

## 程式碼最低要求

每個正式腳本應該說明：

- 目的：這段程式解決什麼論文問題？
- 輸入：讀取哪些資料或參數？
- 輸出：產生哪些結果、表格或圖？
- 執行方式：用什麼命令可以重複執行？
- 對應章節：支撐論文哪一章或哪個研究問題？

## 命名建議

- 資料處理：`prepare_data.*`、`clean_data.*`
- 實驗執行：`run_experiment.*`
- 評估：`evaluate.*`
- 圖表：`make_figures.*`
- 共用工具：`utils.*`

## 從 notebook 移到 src 的標準

符合以下條件時，應該把 notebook 裡的程式整理到 `src/`：

- 需要重複執行；
- 會影響論文結果；
- 教授或口試委員可能要求說明；
- 需要讓其他人重現；
- 已經不是單純探索。
