# 可重複研究檢查清單

這份清單用來確保論文中的程式、資料與實驗結果可以被追蹤與重現。

## 資料

- [ ] 原始資料放在 `data/raw/`，且不直接覆蓋。
- [ ] 清理後資料放在 `data/processed/`。
- [ ] 已記錄資料來源。
- [ ] 已記錄資料取得日期。
- [ ] 已記錄資料是否可公開。
- [ ] 已記錄資料清理規則。

## 程式碼

- [ ] 正式程式放在 `src/`。
- [ ] 探索性 notebook 放在 `notebooks/`。
- [ ] 重要 notebook 邏輯已整理成可重複執行的程式。
- [ ] 每個主要腳本都有目的、輸入、輸出與執行方式。
- [ ] 已記錄需要的套件、框架或工具版本。

## 實驗

- [ ] 每次實驗都有紀錄。
- [ ] 已記錄實驗資料版本。
- [ ] 已記錄模型、方法或演算法設定。
- [ ] 已記錄參數與隨機種子。
- [ ] 已記錄執行命令。
- [ ] 結果放在 `results/`。
- [ ] 圖表放在 `figures/`。

## 論文寫作

- [ ] 論文中的每個重要結果都能對應到實驗紀錄。
- [ ] 論文中的每個重要主張都能對應到文獻、資料或實驗。
- [ ] 圖表有清楚標題、說明與資料來源。
- [ ] 方法章節足以讓他人理解如何重現。
- [ ] 限制與未來工作已誠實說明。

## 交付前

- [ ] 能從 README 找到專案入口。
- [ ] 能找到論文計畫。
- [ ] 能找到主要程式碼。
- [ ] 能找到資料說明。
- [ ] 能找到實驗結果。
- [ ] 能找到文獻筆記。
