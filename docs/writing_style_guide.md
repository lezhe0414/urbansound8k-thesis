# 論文寫作風格指南

這份文件用來維持論文語氣、格式、術語與 AI Agent 修改風格的一致性。若學校或教授提供正式格式規範，應以正式規範為準。

## 預設語言

- 預設使用繁體中文。
- 學術章節使用正式、客觀、清楚的語氣。
- 避免口語化、誇張或沒有證據的描述。
- 英文專有名詞第一次出現時，可使用「中文（English）」格式，之後依 `docs/glossary.md` 統一。

## 語氣原則

- 使用中性、可驗證的表述。
- 將推論、假設、結果與限制清楚區分。
- 不把尚未驗證的內容寫成結論。
- 不使用「顯然」、「必然」、「完全解決」等過度肯定語句，除非有充分證據。

## 論文段落建議

每段盡量符合以下結構：

1. 主題句：說明本段重點。
2. 說明或證據：引用文獻、資料、方法或實驗結果。
3. 小結：連回研究問題或本章目的。

## 常用表述

### 研究背景

- 「近年來，......逐漸受到重視。」
- 「然而，既有研究在......方面仍存在限制。」
- 「因此，本研究聚焦於......」

### 研究目的

- 「本研究旨在......」
- 「本研究欲探討......」
- 「本研究希望透過......」

### 方法說明

- 「本研究採用......作為主要方法。」
- 「資料處理流程包含......」
- 「為評估方法效果，本研究使用......作為指標。」

### 結果說明

- 「實驗結果顯示......」
- 「相較於......，本研究方法在......方面呈現......」
- 「此結果可能表示......，但仍需考量......」

### 限制

- 「本研究仍存在以下限制。」
- 「由於資料來源與研究範圍限制，......」
- 「未來研究可進一步......」

## 引用與證據

- 需要文獻支撐的句子，若尚未找到來源，標示 `[citation needed]`。
- 實驗結果必須能對應 `results/`、`figures/` 或實驗紀錄。
- 重要主張應能在 `references/literature_notes.md` 或 `references/citation_tracker.md` 找到依據。
- 不使用無來源的數字、排名、百分比或比較。

## 圖表寫作

圖表說明應包含：

- 圖表呈現什麼。
- 資料或結果來源。
- 讀者應注意的重點。
- 圖表與研究問題的關係。

## AI Agent 改稿原則

AI Agent 修改論文時應：

- 優先保留使用者原意。
- 改善結構、語氣、可讀性與學術表述。
- 標出需要文獻或實驗支撐的內容。
- 不自行加入未確認的研究結果。
- 若新增推論，需明確標示為待確認。
- 重要修改應更新 `docs/research_log.md`。

## 待教授確認

- 正式論文語言。
- 學校指定格式。
- 引用格式。
- 章節名稱。
- 是否需要 AI 使用揭露。
