# 風險與阻塞追蹤

這份文件用來追蹤可能影響論文進度、程式實作、資料取得、實驗結果與交付時程的風險。風險不是問題本身，而是「如果不處理，可能造成延誤或品質下降」的事項。

## 風險等級

- 高：若不處理，會直接阻礙論文或程式進展。
- 中：會造成延誤或品質問題，但仍可繞過。
- 低：需要注意，但短期內不影響主要進度。

## 狀態

- 開放：尚未解決。
- 監控：暫時可接受，但需持續觀察。
- 已緩解：已採取措施降低風險。
- 已關閉：風險已不存在或已解決。

## 每週檢查問題

- 本週是否出現新的風險？
- 最高等級風險是否已有下一步？
- 是否有風險需要詢問教授？
- 是否有風險需要調整論文題目、程式任務或資料來源？
- 是否有風險已經解決，可以關閉？

## 處理原則

- 高風險事項應優先於一般整理工作。
- 風險的下一步應具體，可在一週內執行或確認。
- 若風險來自教授要求不明，應更新 `docs/professor_questions.md`。
- 若風險影響里程碑，應更新 `docs/milestones.md`。
- 若風險影響研究方向或方法，應更新 `docs/decision_log.md`。
