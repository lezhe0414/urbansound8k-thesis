# 論文章節草稿

這個資料夾用來放各章節的正式草稿。`docs/thesis_outline.md` 是總大綱，這裡則是逐章撰寫與修改的位置。

## 建議使用方式

1. 先在 `docs/thesis_plan.md` 確認題目、研究問題與方法。
2. 用 `docs/thesis_outline.md` 決定章節架構。
3. 在本資料夾逐章撰寫。
4. 每次依教授意見修改後，更新 `docs/meeting_notes.md` 與 `docs/research_log.md`。
5. 章節中的實驗結果、圖表與文獻引用，需能追溯到 `results/`、`figures/`、`references/` 或 `docs/experiment_template.md`。

## 章節檔案

- `00_abstract.md`：摘要
- `01_introduction.md`：緒論
- `02_literature_review.md`：文獻探討
- `03_methodology.md`：研究方法
- `04_system_or_experiment_design.md`：系統設計或實驗設計
- `05_results_and_discussion.md`：結果與討論
- `06_conclusion.md`：結論與未來工作
- `appendix.md`：附錄

## 寫作原則

- 不確定的內容標示為「待確認」。
- 需要引用的句子標示文獻來源或暫記 `[citation needed]`。
- 實驗結果不可憑空填寫，必須對應 `results/` 或實驗紀錄。
- 圖表要能對應 `figures/` 的檔案與圖說。
