# AI Agent 協作流程

這份文件定義之後如何用 AI Agent 協助論文寫作與程式碼開發。

## 每次開始任務時

請盡量提供以下資訊，能提供多少就提供多少：

- 目前要處理的目標
- 相關檔案或資料位置
- 教授的要求或限制
- 希望輸出的形式，例如論文章節、程式、表格、圖、簡報或摘要
- 截止時間

如果資訊不足，AI Agent 應先檢查現有檔案，再提出最少量、最關鍵的確認問題。

## 常見任務類型

### 論文寫作

可請 AI Agent 協助：

- 整理研究背景
- 建立論文章節大綱
- 改寫段落，使語氣更正式
- 檢查論點是否連貫
- 依教授意見修改內容
- 將實驗結果轉成論文敘述

產出應放在 `docs/`。

### 文獻整理

可請 AI Agent 協助：

- 摘要論文
- 比較不同方法
- 整理可引用觀點
- 找出研究缺口
- 建立文獻矩陣

筆記應放在 `references/literature_notes.md` 或 `references/` 內的新檔案。

### 程式碼與實驗

可請 AI Agent 協助：

- 建立資料處理流程
- 實作模型或分析方法
- 建立實驗腳本
- 產生圖表與表格
- 修錯與整理程式碼
- 撰寫執行說明

穩定程式應放在 `src/`，探索性實驗可放在 `notebooks/`。

### 進度管理

每次有明確進度或決策，應更新 `docs/research_log.md`，包含：

- 日期
- 做了什麼
- 得到什麼結果
- 遇到什麼問題
- 下一步是什麼

## 建議提問範例

```text
請幫我把 docs/thesis_plan.md 裡的研究背景改寫成正式論文語氣。
```

```text
請根據 references/ 裡的三篇文獻，整理一個比較表，並指出我的研究可以切入的缺口。
```

```text
請幫我在 src/ 寫一個資料清理腳本，輸入 data/raw/xxx.csv，輸出 data/processed/cleaned.csv。
```

```text
教授說我的方法章節不夠清楚，請幫我看 docs/method.md 並提出修改。
```

## 品質要求

- 寫作內容要能追溯到資料、文獻或實驗結果。
- 程式碼要能重複執行，避免只有一次性的手動操作。
- 實驗結果要留下輸出檔或紀錄。
- 不確定的地方要標示為待確認，不要當成已證實事實。
