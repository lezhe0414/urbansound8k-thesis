# 資料目錄

`data/` 用來放論文使用的資料。資料是否可公開、是否能進 Git、是否需要匿名化，需依資料來源與教授要求確認。

## 結構

- `raw/`：原始資料，不直接修改或覆蓋。
- `processed/`：清理、轉換、篩選或標註後的資料。

## 資料紀錄最低要求

每份重要資料至少記錄：

- 資料名稱：
- 資料來源：
- 取得日期：
- 授權或使用限制：
- 是否可公開：
- 欄位說明：
- 清理方式：
- 對應程式：
- 對應實驗：

## 原則

- 不直接修改 `data/raw/` 裡的原始資料。
- 清理後資料放在 `data/processed/`。
- 大型資料不一定適合放進 Git，需另行記錄儲存位置。
- 如果資料包含個資或敏感資訊，必須先匿名化或依學校規範處理。
